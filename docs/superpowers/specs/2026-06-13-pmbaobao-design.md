## 目标

在本地开发一个前后端完整的 `pmbaobao.com` 首页导航站：

- 前端尽量还原原站首页导航体验。
- 后端提供本地 API。
- 数据库使用本地 SQLite。
- 数据从 `pmbaobao.com` 首页公开可见 HTML 中抓取并入库。

本项目不会连接、模拟或破解原站私有后端，也不会获取非公开数据。所谓“用他的数据”限定为：抓取公开首页中能直接看到的分类、站点卡片、描述、外链和图标等内容。

## 范围

本期包含：

## 推荐技术方案

使用一个 Next.js 全栈项目：

- Next.js App Router：负责前端页面和后端 API。
- TypeScript：保证数据结构清晰。
- SQLite：本地数据库。
- Prisma：数据库建模、迁移和访问。
- Cheerio：解析抓取到的 HTML。
- 自写 CSS：还原 OneNav 风格布局，不直接复制原站主题源码。

选择原因：

- 一个项目即可完成前后端。
- SQLite 不需要额外安装数据库服务。
- Prisma 方便维护表结构和数据同步逻辑。
- Next.js API Routes 足够支撑这个项目的后端需求。

## 数据模型

### `categories`

保存侧边栏分类、Tab 分类和普通区块分类。

字段：

- `id`：本地主键。
- `sourceId`：原站分类 id，例如 `1174`。
- `parentSourceId`：原站父分类 id，没有则为空。
- `name`：分类名称。
- `slug`：本地生成的稳定标识。
- `iconClass`：原站侧边栏图标 class，能解析到就保存。
- `kind`：分类类型，可选值为 `sidebar`、`tab`、`section`。
- `sortOrder`：排序。

### `sites`

保存站点卡片。

字段：

- `id`：本地主键。
- `sourceId`：原站站点 id，来自 `data-id`。
- `sourceDetailUrl`：原站详情页地址，例如 `https://www.pmbaobao.com/sites/3047.html`。
- `title`：站点名称。
- `description`：站点描述。
- `outboundUrl`：真实外链地址，来自 `data-url`。
- `iconUrl`：图标地址。
- `isNoContentCard`：是否属于原站的紧凑卡片样式。
- `sortOrder`：排序。

### `site_categories`

保存站点和分类的关联关系。

字段：

- `siteId`：站点 id。
- `categoryId`：分类 id。
- `sortOrder`：站点在该分类下的排序。

### `sync_runs`

保存每次抓取同步的结果。

字段：

- `id`
- `sourceUrl`
- `status`
- `startedAt`
- `finishedAt`
- `categoryCount`
- `siteCount`
- `message`

## 抓取策略

抓取脚本流程：

1. 请求 `https://www.pmbaobao.com/`，使用普通浏览器 User-Agent。
2. 把原始 HTML 快照保存到本地缓存目录，便于调试。
3. 从 `.sidebar-item` 解析侧边栏主分类。
4. 从 `.parent-category` 和附近的 `.slider_menu` 解析 Tab 分类组。
5. 从 `.url-card a.card` 解析站点卡片。
6. 对每个卡片提取：
   - `data-id` 作为原站站点 id。
   - `href` 作为原站详情页地址。
   - `data-url` 作为真实外链地址。
   - 卡片可见文本作为站点名称。
   - `title` 或描述段落作为站点描述。
   - `img src` 作为图标地址。
7. 根据卡片所在的最近分类区块或 Tab 面板建立分类关系。
8. 写入 SQLite，重复同步时使用 upsert 更新已有数据。

已观察到的公开页面结构：

- 首页 HTML 中约有 248 个站点卡片。
- 侧边栏主分类约 14 个。
- Tab 面板约 19 个。
- 部分 Tab 是懒加载占位，HTML 里没有真实卡片。对这类 Tab，本地保留分类结构，并显示空状态。

## 后端接口

### `GET /api/categories`

返回侧边栏分类、Tab 分类和层级关系。

### `GET /api/sites`

查询站点卡片。

支持参数：

- `category`：分类 id 或 slug。
- `q`：关键词。
- `limit`：分页数量。
- `offset`：分页偏移。

### `GET /api/search`

本地搜索。

支持参数：

- `q`：搜索关键词。

搜索范围：

- 站点名称
- 描述
- 外链地址

### `GET /api/sync`

返回最近一次同步状态和当前数据库统计。

### `POST /api/sync`

手动触发一次公开首页重新抓取。

## 前端设计

首页就是导航工具本身，不做营销落地页。

布局：

- 桌面端左侧固定侧边栏，宽度约 200px。
- 移动端侧边栏可收起。
- 顶部为 Logo、搜索框、搜索引擎切换和主题切换。
- 主内容区保持原站类似的信息密度。
- 卡片使用小圆角、紧凑间距、左图标右标题描述的布局。
- 分类顺序尽量保持和原站一致。
- 有子分类的区块显示横向 Tab。

交互：

- 点击侧边栏滚动到对应分类。
- 点击 Tab 切换对应分类下的卡片。
- 搜索框支持本地筛选，也可按当前搜索引擎打开外部搜索。
- 点击站点卡片在新标签页打开真实外链。
- 明暗主题切换保存在本地。
- 同步状态以一个较轻的开发信息区域展示，不做大面积说明卡片。

视觉目标：

- 还原原站 OneNav 风格的信息密度、侧边栏、浅灰背景、卡片网格和导航节奏。
- 不直接复制原站主题源码。
- 使用本地 CSS 根据公开页面观察重新实现。

## 错误处理

- 抓取失败时保留数据库里已有数据，并把失败信息写入 `sync_runs`。
- 图标加载失败时显示首字母或默认图标。
- 分类没有卡片时显示紧凑空状态。
- 外链格式异常时仍显示卡片，但禁用打开操作，并记录解析问题。

## 验证标准

最低验证要求：

- 抓取脚本能用保存的 HTML fixture 跑通。
- 数据库同步命令能完成，并输出分类和站点数量。
- API 冒烟测试通过：
  - 分类接口
  - 站点列表接口
  - 搜索接口
  - 同步状态接口
- 前端构建通过。
- 使用浏览器检查桌面和移动端：
  - 侧边栏正常显示。
  - 搜索可用。
  - 分类和站点卡片来自数据库。
  - 文本没有明显重叠。
  - 明暗主题切换可用。
